# Sprint 0.5 — Frontend Foundation (API Client + Auth Hook)

**Date**: 2026-07-03

## Problem

The frontend web application needed:
1. A robust API client that injects authorization headers, handles response unwrapping/error normalization, and supports automatic token refresh on 401 response (silent retry).
2. A global authentication context (`AuthProvider` and `useAuth()`) to store user details, tokens, and manage logins/logouts.
3. Proper Next.js layout wrapping and top navbar integration (replacing the hardcoded profile with actual user session details or a premium Log In button).
4. Unit tests verifying all API client requirements.

---

## Solution

We implemented a custom, zero-dependency, fetch-based API client coupled with a React authentication context and a premium glassmorphic modal interface.

### Files Created

1. **`apps/web/src/lib/api-client.ts`** — Implements `apiClient` using standard browser `fetch`:
   - Configures the base URL from `process.env.NEXT_PUBLIC_API_URL` (falls back to localhost).
   - Injects `Authorization: Bearer <token>` from `localStorage` safely in client-side environments.
   - Dedupes simultaneous refresh requests using a shared `refreshPromise` and triggers a silent retry on 401.
   - Normalizes errors into custom `ApiError` instances and unwraps successful `{ data }` envelopes.
   - Registers pre-populated logging interceptors for request/response metrics.
2. **`apps/web/src/lib/auth-context.tsx`** — Provides global `AuthContext` state (`user`, `token`, `isAuthenticated`) persisting authentication data inside `localStorage` and syncing with API client session resets. Fallback-decodes the JWT token for profile data.
3. **`apps/web/src/lib/api-client.test.ts`** — A complete test suite running via the native Node.js test runner verifying base URL construction, auth header injection, response unwrapping, error normalization, and 401 silent retry logic.
4. **`apps/web/.env`** — Specifies `NEXT_PUBLIC_API_URL=http://localhost:3000/api/v1` for the web client.

### Files Modified

1. **`apps/web/src/app/layout.tsx`** — Wrapped the Next.js App Router root layout children in the `<AuthProvider>`.
2. **`apps/web/src/components/home/top-navbar.tsx`** — Replaced the hardcoded profile with conditional rendering:
   - Authenticated: Shows user initials avatar, name, and a "Log Out" dropdown on hover.
   - Unauthenticated: Shows a "Log In" button.
   - Sleek auth modal with Login, Register, and a one-click **"Quick Test Login"** button which logs in (or registers) a predefined backend test user (`test@wrectifai.com`).
3. **`apps/web/package.json`** — Added `"test"` run script utilizing native Node.js test runner and `@swc-node/register`.
4. **`apps/web/project.json`** — Registered the `"test"` target in Nx targets list, and configured the `dev` and `start` commands to bind to port `3001` (preventing port collisions with the API server on port `3000`).
