## Deployment Steps

### Step 1: Push to GitHub
```bash
git add -A
git commit -m "feat: add Render deployment config with static Next.js export"
git push
```

### Step 2: Create Render Account & Blueprint
1. Go to [render.com](https://render.com) → Sign up / Log in
2. Click **New Blueprint** (or **New +** → **Blueprint**)
3. Connect your GitHub repo
4. Render will detect `render.yaml` and auto-create all 3 services

### Step 3: Set Secrets (after first deploy)
Go to each service's **Environment** tab and set:

**API service (`wrectifai-api`):**
- `GROQ_API_KEY` — your Groq API key (paste the real value)
- `NEXT_PUBLIC_GOOGLE_CLIENT_ID` — your Google OAuth client ID

**Web service (`wrectifai-web`):**
- `NEXT_PUBLIC_GOOGLE_CLIENT_ID` — same Google OAuth client ID

`JWT_SECRET` and `JWT_REFRESH_SECRET` are auto-generated by Render.

### Step 4: Run Database Migrations
After the Postgres service is live:
```bash
# Get the connection string from Render Dashboard → wrectifai-db → Connect → Internal URL
psql "postgresql://user:password@host:5432/wrectifai" \
  -f db/migrations/001_initial_schema.sql \
  -f db/migrations/002_refresh_tokens.sql \
  -f db/migrations/003_add_vehicle_soft_delete.sql \
  -f db/migrations/004_known_issues.sql \
  -f db/migrations/005_dummy_test_user.sql \
  -f db/migrations/006_promos.sql \
  -f db/migrations/007_quotes.sql \
  -f db/migrations/008_bookings_seed.sql \
  -f db/migrations/009_update_garages_rich_meta.sql \
  -f db/migrations/010_remove_garages_ui_columns.sql \
  -f db/migrations/011_add_garages_response_mins.sql \
  -f db/migrations/012_add_missing_garages.sql \
  -f db/migrations/013_diagnose_ui_config.sql \
  -f db/migrations/014_diagnose_ui_config_seed.sql
```

### Step 5: Update CORS
Once deployed, update `WEB_ORIGINS` in the API service to include the actual web URL:
```
https://wrectifai-web.onrender.com
```
